# Project code structure — Countdown

This document describes the folder layout and responsibilities of the main Dart files in the repository. Use it as a quick reference before moving on to platform work (Android).

## Top-level folders

- `lib/` — application source code (UI, screens, services, models, theming, localization).
- `test/` — unit/widget tests.
- `android/`, `ios/`, `windows/` — platform-specific projects and native integration.
- `build/` — build outputs (generated by Flutter).

---

## `lib/` (core application)

- `lib/main.dart` — App entry point. Initializes `SharedPreferences`, services (notifications), theme and locale, and constructs the `CountdownApp` widget.
- `lib/theme_manager.dart` — Defines app theme configuration, maps `AppThemeMode` to `ThemeData` and centralizes ColorScheme usage across the app.

### Localization
- `lib/l10n/strings.dart` — Main localization wrapper (`S`) selecting RU/EN strings used in UI.
- `lib/l10n/strings_plurals.dart` — Pluralization helpers and localized message variants (recreated with proper encoding during fixes).

### Models
- `lib/models/day_usage.dart` — Small model representing usage for one day: `{ date, count }`.
- `lib/models/usage_stats.dart` — Aggregated usage statistics computed from stored timestamps: total count, per-day `DayUsage` list, streak calculation, badge key, helpers like `getTodayCount()`, `getYesterdayCount()`, `getMaxDayCount()`.

### Screens
- `lib/screens/home_screen.dart` — The main app UI: layout for timer, control buttons, top menus, and embedding child widgets. Wires user interaction to `HomeTimerController`.
- `lib/screens/analytics_screen.dart` — Analytics / statistics UI showing historical usage charts and badges.
- `lib/screens/home_timer_controller.dart` — Core timer business logic and state (`HomeTimerController`): bootstrap/restore timer, start/pause/reset/sync with system clock, track `TimerState`, handle completion, record usages in `UsageStore`, trigger notifications, and run milestone/achievement checks. This file contains the achievement deduplication logic added recently.

### Services
- `lib/services/notification_service.dart` — Cross-platform façade for in-app notifications. Routes calls to platform handlers and provides helpers: `init()`, `requestPermission()`, `scheduleSteps(...)` (schedule mid/near-end notifications), `cancelSteps()`, `showDone()`, `showBadge()`, `showAchievement()`.
- `lib/services/_mobile_notification_handler.dart` — Mobile implementation using `flutter_local_notifications` + timezone handling. Schedules native notifications and handles permission requests for Android/iOS.
- `lib/services/_desktop_notification_handler.dart` — Desktop implementation using `local_notifier` and timers to simulate scheduled notifications on desktop platforms.
- `lib/services/notification_prefs.dart` — Wrapper around `SharedPreferences` for notification user preferences: sound/vibration and (new) last shown achievement key + timestamp for deduplication.
- `lib/services/timer_prefs.dart` — Persisted timer preferences: saved minutes and `endsAt` timestamp; helpers to load/save/clear.
- `lib/services/usage_store.dart` — Persist and load timestamped usage events using `SharedPreferences`. Produces `UsageStats` via `UsageStats.fromTimestamps()`.
- `lib/services/theme_prefs.dart` — Persist user theme selection (`AppThemeMode`) and load/save helpers.

### UI primitives
- `lib/ui/circular_timer_button.dart` — Timer button widget and visuals (progress ring, resurface/fade on completion). Uses `Theme.of(context).colorScheme` for colors.
- (Other UI components such as ring-based timers or glow effects used during earlier UI work are located under `lib/ui/`.)

### Widgets
- `lib/widgets/settings_sheet.dart` — Modal sheet with settings controls (theme, language, notification toggles). Uses `NotificationPrefs` and `ThemePrefs`.
- `lib/widgets/timer_sheet.dart` — Modal for adjusting timer minutes, sliders and quick presets.
- `lib/widgets/dot_menu_button.dart` — Small overflow menu button used in top corner menus.
- `lib/widgets/expandable_corner_menu.dart` — Corner menu with expandable action buttons for settings, stats etc.
- `lib/widgets/statistics_icon_button.dart` — Icon button that navigates to analytics/statistics view.
- `lib/widgets/corner_icon_button.dart` — Reusable icon button for top corners with consistent theme colors.
- `lib/widgets/control_panel.dart` — Playback controls (start/pause/reset) and auxiliary controls; adjusted to avoid overflow.

## `test/` (automated tests)

- `test/widget_test.dart` — Basic widget smoke test that boots `HomeScreen` and verifies the timer text and UI presence.
- `test/usage_stats_test.dart` — Unit tests for usage statistics logic (if present).
- `test/home_timer_controller_achievement_test.dart` — Tests added to validate achievement notification logic and deduplication. Uses mocked/shared-preferences-backed helpers for verification.

---

## Platform folders
- `android/` — Android Gradle project: manifests, Gradle scripts, `app/` module. For Android build and notification channel configuration this directory will be updated when preparing the Android target.
- `ios/` — Xcode project for iOS; iOS notification permissions are coordinated via the mobile notification handler.
- `windows/` — Native runner and Flutter embedding for Windows. Release exe is produced here at `build/windows/x64/runner/Release/countdown.exe` after `flutter build windows --release`.

---

## Notes & pointers
- The canonical app initialization happens in `lib/main.dart`: it wires `NotificationService.init()` and `requestPermission()`, creates `NotificationPrefs`, `TimerPrefs`, `UsageStore`, then passes them to `HomeScreen`.
- Notification flow summary:
  - `HomeTimerController.start()` and `bootstrap()` call `NotificationService.scheduleSteps(...)` to schedule end/halfway/10s reminders.
  - On completion `HomeTimerController` calls `notifications.showDone()` and then milestone/achievement logic that uses `NotificationPrefs` to dedupe `showAchievement()` calls.
- Theme & color decisions centralized in `lib/theme_manager.dart` and `ThemePrefs`.
- Before moving to Android work, consider:
  - Verifying Android notification channel settings and runtime permission flows in `lib/services/_mobile_notification_handler.dart` and `android/` manifests.
  - Reviewing translated strings in `lib/l10n/*` for new achievement messages.
